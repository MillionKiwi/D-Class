디클래스(D-Class) UI 설계 명세서 v2.0
📋 문서 정보
* 프로젝트명: D-Class (디클래스)
* 문서 버전: v2.0
* 작성일: 2025년 10월
* 플랫폼: Mobile-first PWA (Progressive Web App)
* 프로젝트 비전: 무용 강사와 학원을 위한 신뢰 기반의 투명하고 효율적인 구인/구직 통합 플랫폼
________________


🎨 디자인 시스템
1. 색상 팔레트 (Color Palette)
Primary Color (메인 색상)
* Serene Blue: #A7C7E7 (신뢰감, 안정감)
* 역할: 헤더, 주요 아이콘, 선택된 요소(필터, 탭), 포커스 상태
Accent Color (포인트 색상)
* Soft Coral: #FAB4B4 (행동 유도, 주목)
* 역할: '지원하기', '등록하기' 등 핵심 CTA 버튼
Neutral Colors (중립 색상)
* Background: #F9F9F9 (기본 배경)
* Card Background: #FFFFFF (카드 배경)
* Primary Text: #5D6D7E (기본 텍스트)
* Secondary Text: #AEB6BF (보조 텍스트)
* Divider: #EAEAEA (구분선)
* Disabled: #D5DBDB (비활성 상태)
Semantic Colors (의미별 색상)
* Success (성공, 인증): #B2D8D8 (파스텔 민트)
* Warning (주의, 대기): #FFE5A9 (파스텔 옐로우)
* Error (오류, 반려): #FFB6C1 (파스텔 핑크)
* Info (정보): #D4E5F7 (연한 블루)
2. 타이포그래피 (Typography)
* 폰트 패밀리: Pretendard
* 기본 스케일:
   * H1 (페이지 제목): 22px / Bold / Line-height 1.4
   * H2 (섹션 제목): 18px / SemiBold / Line-height 1.4
   * H3 (서브 섹션): 16px / SemiBold / Line-height 1.4
   * Body (본문): 16px / Regular / Line-height 1.6
   * Body Small (작은 본문): 14px / Regular / Line-height 1.5
   * Caption (부가 정보): 12px / Regular / Line-height 1.4
3. 공통 컴포넌트 (Core Components)
버튼 (Buttons)
* Primary Button: Soft Coral 배경, 흰색 텍스트, 8px 모서리 둥글기, 높이 48px
* Secondary Button: 흰색 배경, Serene Blue 테두리 및 텍스트, 8px 모서리 둥글기
* Text Button: 배경 없음, Serene Blue 텍스트
* Disabled State: Disabled 색상 배경, 투명도 50%
입력 필드 (Input Fields)
* 기본 상태: 흰색 배경, #EAEAEA 테두리, 8px 모서리 둥글기, 높이 48px
* 포커스 상태: Serene Blue 테두리 (2px)
* 에러 상태: Error 색상 테두리, 하단에 에러 메시지 표시
* 성공 상태: Success 색상 테두리
카드 (Cards)
* 배경: 흰색
* 모서리: 12px 둥글기
* 그림자: 0px 2px 8px rgba(0, 0, 0, 0.08)
* 패딩: 16px
* 간격: 카드 간 12px
배지 (Badges)
* 인증 배지: Success 배경, 흰색 체크 아이콘, 12px 모서리 둥글기, 작은 사이즈
* 상태 배지: 상태별 Semantic Color 배경, 흰색 텍스트, 완전 둥근 모서리
* 개수 배지: Soft Coral 배경, 흰색 텍스트, 원형
모달 & 팝업
* 배경 딤: rgba(0, 0, 0, 0.5)
* 모달: 흰색 배경, 16px 모서리 둥글기, 상단 타이틀, 중앙 내용, 하단 액션 버튼
* 바텀시트: 화면 하단에서 올라오는 형태, 상단 핸들 바 포함
토스트 (Toast)
* 위치: 화면 하단에서 56px 위 (네비게이션 바 위)
* 스타일: 어두운 배경 (rgba(0, 0, 0, 0.8)), 흰색 텍스트
* 지속 시간: 3초
* 애니메이션: 페이드 인/아웃
로딩 인디케이터
* 풀스크린 로딩: 중앙에 Serene Blue 스피너
* 인라인 로딩: 해당 영역에 작은 스피너
* 스켈레톤: 카드 형태의 로딩 플레이스홀더
빈 상태 (Empty State)
* 아이콘: 연한 회색 일러스트레이션
* 메시지: Primary Text 색상
* CTA: 있는 경우 Secondary Button으로 표시

📱 페이지 상세 명세
P1. 인증 및 회원
P1-1. 로그인 페이지
🎯 페이지 목표: 기존 사용자가 빠르고 간편하게 서비스에 접근
📐 레이아웃:
* 상단: 로고 및 서비스 소개 문구
* 중앙: 이메일, 비밀번호 입력 필드
* 로그인 버튼 (Primary)
* 하단: 회원가입 링크, 비밀번호 찾기 링크
🎨 디자인 상세:
* 입력 필드 포커스 시 Serene Blue 테두리
* 로그인 버튼은 Soft Coral
* 링크는 Serene Blue Text Button
🔄 인터랙션:
* 로그인 실패 시: Error 색상 메시지 "이메일 또는 비밀번호가 올바르지 않습니다"
* 로딩 중: 버튼 비활성화 및 로딩 스피너 표시
* 성공 시: 역할에 따라 해당 메인 페이지로 리다이렉트
⚠️ 에러 처리:
* 네트워크 오류: "네트워크 연결을 확인해주세요" 토스트
* 서버 오류: "일시적인 오류가 발생했습니다" 토스트
________________


P1-2. 회원가입 - 역할 선택
🎯 페이지 목표: 사용자가 자신의 역할을 명확하게 인지하고 선택
📐 레이아웃:
* 상단: 진행 단계 인디케이터 (1/3)
* 안내 문구: "어떤 유형으로 가입하시나요?"
* 중앙: 2개의 선택 카드 (강사 / 학원)
   * 각 카드: 아이콘, 역할명, 간단한 설명
* 하단: [다음] 버튼 (초기 비활성화)
🎨 디자인 상세:
* 선택되지 않은 카드: 흰색 배경, 연한 테두리
* 선택된 카드: Serene Blue 테두리 (2px), 배경색 약간 변화
* [다음] 버튼: 선택 전 Disabled, 선택 후 Soft Coral로 활성화
🔄 인터랙션:
* 카드 탭 시: 선택 효과 + [다음] 버튼 활성화
* [다음] 탭: 다음 단계(P1-3)로 이동
________________


P1-3. 회원가입 - 정보 입력
🎯 페이지 목표: 역할에 맞는 필수 정보를 입력받아 회원가입 완료
📐 레이아웃:
* 상단: 진행 단계 인디케이터 (2/3)
* 입력 필드:
   * 이메일 (중복 확인 버튼)
   * 비밀번호 (보안 강도 표시)
   * 비밀번호 확인
   * 이름
   * 연락처
   * [강사] 전문 분야 (다중 선택)
   * [학원] 학원명, 주소 (검색)
* 약관 동의:
   * 전체 동의 체크박스
   * 개별 동의 (필수): 이용약관, 개인정보처리방침
   * 개별 동의 (선택): 마케팅 수신 동의
* 하단: [가입 완료] 버튼
🎨 디자인 상세:
* 필수 항목 옆에 빨간 별표(*) 표시
* 비밀번호 강도: 약함(Error), 보통(Warning), 강함(Success)
* 중복 확인 버튼: Secondary Button
* 약관 텍스트 탭 시 상세 페이지 바텀시트로 표시
🔄 인터랙션:
* 이메일 중복 확인: 실시간 검증, 사용 가능 시 Success 메시지
* 비밀번호 불일치: Error 메시지 표시
* 필수 약관 미동의 시: [가입 완료] 버튼 비활성화
* 가입 완료 시: "가입이 완료되었습니다" 토스트 → 로그인 페이지로 이동
⚠️ 에러 처리:
* 이미 가입된 이메일: "이미 사용 중인 이메일입니다"
* 유효하지 않은 이메일 형식: "올바른 이메일 형식을 입력해주세요"
* 비밀번호 조건 미달: "8자 이상, 영문/숫자/특수문자 포함"
________________


P1-4. 이용약관 페이지
🎯 페이지 목표: 서비스 이용약관을 명확하게 제공
📐 레이아웃:
* 상단: 헤더 (뒤로가기, "이용약관")
* 스크롤 가능한 본문 영역
* 하단: [닫기] 버튼
🎨 디자인 상세:
* 읽기 편한 여백과 줄간격
* 중요 조항은 볼드 처리
________________


P1-5. 개인정보처리방침 페이지
🎯 페이지 목표: 개인정보처리방침을 명확하게 제공
📐 레이아웃: P1-4와 동일한 구조
________________


P2. 강사 회원
P2-1. 메인 대시보드 (공고 목록 및 지도)
🎯 페이지 목표: 구인 공고를 리스트 또는 지도 형태로 효과적으로 탐색하고 필터링
📐 레이아웃:
* 상단 헤더: 로고, 검색, 알림
* 필터링 영역 (가로 스크롤):
  * 지역 (서울/경기/인천 → 시/구)
  * 무용 장르 (발레, 현대무용, 한국무용 등)
  * 근무 시간 (오전/오후/저녁/주말)
* 뷰/정렬 영역:
  * 좌측: 정렬 옵션 (최신순, 급여 높은 순)
  * 우측: 뷰 전환 토글 (아이콘 버튼: 리스트 / 지도)
* 콘텐츠 영역 (뷰 전환에 따라 변경):
  * A. 리스트 뷰 (기본):
    * 공고 카드 목록 (무한 스크롤)
  * B. 지도 뷰:
    * 인터랙티브 지도 영역 (콘텐츠 영역 전체)
    * 지도 위 공고 핀 (클러스터링 지원)
    * 지도 이동 시: [현재 위치에서 재검색] 버튼 노출
    * 핀 선택 시: 하단 '미니 카드 바텀시트' 노출
* 하단 네비게이션 바 (고정)
🎨 디자인 상세:
* 필터 칩: 기본 흰색 배경, 선택 시 **Serene Blue** 배경
* 뷰 전환 토글:
  * 선택된 뷰 (아이콘): **Serene Blue** 배경, 흰색 아이콘 (선택된 필터 칩 스타일 준용)
  * 비선택 뷰 (아이콘): 흰색 배경, **Serene Blue** 아이콘 (**Text Button** 스타일 준용)
* A. 리스트 뷰:
  * 공고 카드: (**Card** 컴포넌트 스타일 준용)
    * 학원명 (인증 배지 표시)
    * 공고 제목
    * 지역, 무용 장르 (아이콘)
    * 급여 (볼드 강조)
    * 등록일
    * 우측 상단: 찜 아이콘 (하트)
  * 이미 지원한 공고: 좌측 상단에 "지원 완료" 배지
* B. 지도 뷰:
  * 지도 핀: **Serene Blue** 색상
  * 핀 클러스터: **Soft Coral** 배경, 흰색 텍스트 ('개수 배지' 스타일 준용)
  * [현재 위치에서 재검색] 버튼: **Secondary Button** 스타일 (흰색 배경, Serene Blue 테두리)
  * 미니 카드 바텀시트: **Card** 컴포넌트 스타일 준용, 하단 네비게이션 바 위에 노출
    * 내용: 학원명, 인증 배지, 공고 제목, 급여, 찜 아이콘
🔄 인터랙션:
* 뷰 전환 토글 탭:
  * 리스트 뷰 <-> 지도 뷰 즉시 전환 (애니메이션 적용)
  * 선택된 뷰 상태는 사용자 세션 동안 유지
* 필터 변경:
  * 리스트 뷰: 목록 즉시 업데이트
  * 지도 뷰: 지도 위의 핀 즉시 업데이트
* A. 리스트 뷰:
  * 카드 탭: 공고 상세(P2-2) 페이지로 이동
  * 찜 아이콘 탭: 찜 추가/제거 (애니메이션)
  * 스크롤 하단 도달: 다음 페이지 로딩 (스피너 표시)
* B. 지도 뷰:
  * 지도 핀 탭: 해당 공고의 '미니 카드 바텀시트' 하단에서 슬라이드 업
  * 미니 카드 바텀시트 탭: 공고 상세(P2-2) 페이지로 이동
  * 지도 이동 (드래그/줌): [현재 위치에서 재검색] 버튼 노출
  * [현재 위치에서 재검색] 탭: 현재 지도 영역 기준으로 공고 핀 갱신 (인라인 로딩)
* 풀 투 리프레시: 목록 또는 지도 핀 새로고침
🔍 빈 상태:
* 조건에 맞는 공고 없음 (리스트/지도 공통): "조건에 맞는 공고가 없습니다" + 필터 초기화 버튼
* 아직 공고가 없음: "곧 새로운 공고가 등록될 예정입니다"
⚠️ 에러 처리:
* 로딩 실패: "공고를 불러오는데 실패했습니다" + 재시도 버튼
________________


P2-2. 공고 상세 조회
🎯 페이지 목표: 구인 공고의 모든 정보를 파악하고 지원 결정
📐 레이아웃:
* 상단 헤더: 뒤로가기, 찜 아이콘
* 학원 정보 섹션:
   * 학원명, 인증 배지
   * 평균 평점 (별 아이콘)
   * 위치 (지도 미리보기 - 클릭 시 확대)
   * [학원 프로필 보기] 버튼
* 모집 요강 섹션:
   * 공고 제목
   * 모집 장르
   * 담당 수업
   * 근무 요일 및 시간
   * 급여 정보
   * 우대 사항
* 상세 설명 섹션
* 하단 고정:
   * [학원에게 문의하기] (Secondary Button)
   * [지원하기] (Primary Button)
🎨 디자인 상세:
* 섹션 간 구분선으로 명확한 분리
* 중요 정보(급여, 시간)는 볼드 및 아이콘 사용
* 이미 지원한 공고: [지원하기] 버튼 → "지원 완료" (Disabled)
🔄 인터랙션:
* [학원 프로필 보기]: P2-9 학원 프로필 페이지로 이동
* [지원하기]: 확인 모달 표시
   * "이 공고에 지원하시겠습니까?"
   * [취소] / [지원하기]
* 지원 완료 후:
   * "지원이 완료되었습니다" 토스트
   * 버튼 상태 변경
   * 지원 현황(P2-5)에 추가
* [문의하기]: 학원 연락처 표시 또는 외부 연락 수단 안내
⚠️ 에러 처리:
* 지원 실패: "지원 처리 중 오류가 발생했습니다" 토스트
________________


P2-3. 검색 페이지
🎯 페이지 목표: 키워드로 공고를 빠르게 찾기
📐 레이아웃:
* 상단: 검색 입력 필드 (자동 포커스), 취소 버튼
* 최근 검색어 (삭제 가능)
* 인기 검색어
* 검색 결과 목록 (입력 시)
🎨 디자인 상세:
* 검색어는 Serene Blue 칩으로 표시
* 검색 결과: P2-1과 동일한 카드 형태
🔄 인터랙션:
* 입력 시 실시간 검색 (debounce 적용)
* 최근 검색어 탭: 자동 입력
* 검색 결과 카드 탭: 공고 상세로 이동
________________


P2-4. 마이페이지
🎯 페이지 목표: 개인 정보 및 활동 관리의 허브
📐 레이아웃:
* 상단: 프로필 요약
   * 프로필 사진
   * 이름, 인증 배지
   * [프로필 관리] 버튼
* 메뉴 목록:
   * 내 활동
      * 지원 현황
      * 찜한 공고
      * 내가 작성한 리뷰
   * 인증 관리
      * 학력/경력 인증 (상태 표시)
   * 설정
      * 알림 설정
      * 계정 설정
      * 고객센터
      * 로그아웃
🎨 디자인 상세:
* 각 메뉴: 아이콘 + 텍스트 + 우측 화살표
* 인증 상태:
   * 미인증: Warning 배지
   * 인증 완료: Success 배지
   * 인증 대기: Info 배지
🔄 인터랙션:
* 각 메뉴 탭 시 해당 페이지로 이동
________________


P2-5. 프로필 관리
🎯 페이지 목표: 강사의 전문성을 어필하는 프로필 작성 및 수정
📐 레이아웃:
* 상단 헤더: 뒤로가기, "프로필 관리", [저장] 버튼
* 프로필 사진 변경 영역 (카메라 아이콘)
* 기본 정보:
   * 이름 (수정 불가, 회색 처리)
   * 연락처
   * 이메일 (수정 불가)
* 전문 분야 (무용 장르, 다중 선택 칩)
* 자기소개 (텍스트 영역, 최대 500자)
* 경력 목록:
   * 각 경력: 카드 형태
   * [+ 경력 추가] 버튼
* 학력 목록:
   * 각 학력: 카드 형태
   * [+ 학력 추가] 버튼
🎨 디자인 상세:
* 경력/학력 카드: 기관명, 기간, 설명, 우측 상단에 수정/삭제 아이콘
* 입력 필드 카운터 표시 (예: 325/500)
🔄 인터랙션:
* 프로필 사진 탭: 이미지 선택 바텀시트 (카메라/갤러리)
* [+ 경력/학력 추가]: 입력 모달 표시
* 수정 아이콘: 해당 항목 수정 모달
* 삭제 아이콘: 삭제 확인 모달
* [저장]: 변경 사항이 있을 때만 활성화
   * 저장 완료 시 "프로필이 저장되었습니다" 토스트
* 뒤로가기 (변경사항 있을 시): "저장하지 않은 변경사항이 있습니다" 확인 모달
⚠️ 에러 처리:
* 이미지 업로드 실패: "이미지 업로드에 실패했습니다" 토스트
* 저장 실패: "저장 중 오류가 발생했습니다" 토스트
________________


P2-6. 지원 현황
🎯 페이지 목표: 지원한 공고의 상태를 쉽게 확인
📐 레이아웃:
* 상단 헤더: 뒤로가기, "지원 현황"
* 상태별 탭: 전체, 검토중, 최종합격, 불합격
* 지원 공고 카드 목록:
   * 학원명, 인증 배지
   * 공고 제목
   * 지원일
   * 우측 상단: 상태 배지
🎨 디자인 상세:
* 탭: 선택된 탭 Serene Blue 하단 바
* 상태 배지:
   * 검토중: Warning
   * 최종합격: Success
   * 불합격: Error
🔄 인터랙션:
* 탭 전환: 해당 상태 공고만 필터링
* 카드 탭: 공고 상세로 이동
* 최종합격 상태 카드: [리뷰 작성하기] 버튼 표시 (미작성 시)
🔍 빈 상태:
* "아직 지원한 공고가 없습니다" + [공고 둘러보기] 버튼
________________


P2-7. 찜한 공고
🎯 페이지 목표: 관심 있는 공고를 모아서 관리
📐 레이아웃:
* 상단 헤더: 뒤로가기, "찜한 공고"
* 공고 카드 목록 (P2-1과 동일한 형태)
🔄 인터랙션:
* 카드 탭: 공고 상세로 이동
* 찜 아이콘 탭: 목록에서 제거 (확인 없이 즉시)
🔍 빈 상태:
* "찜한 공고가 없습니다" + [공고 둘러보기] 버튼
________________


P2-8. 인증 서류 제출
🎯 페이지 목표: 학력/경력 증명 서류 업로드로 신뢰도 향상
📐 레이아웃:
* 상단 헤더: 뒤로가기, "학력/경력 인증"
* 현재 인증 상태 카드:
   * 미인증 / 인증 대기 / 인증 완료 / 인증 반려
* 안내 문구:
   * "졸업증명서, 경력증명서 등을 업로드해주세요"
   * 허용 파일: JPG, PNG, PDF (최대 10MB)
* 파일 첨부 영역 (점선 박스, 업로드 아이콘)
* 첨부된 파일 목록 (파일명, 크기, 삭제 버튼)
* 하단: [인증 신청하기] 버튼
🎨 디자인 상세:
* 상태별 색상:
   * 미인증: Disabled
   * 인증 대기: Warning
   * 인증 완료: Success
   * 인증 반려: Error (반려 사유 표시)
* 파일 첨부 영역: 탭 가능한 점선 박스
🔄 인터랙션:
* 첨부 영역 탭: 파일 선택
* 파일 선택 후: 목록에 추가
* 삭제 버튼: 목록에서 제거
* [인증 신청하기]: 파일 1개 이상일 때 활성화
   * 신청 완료 시 "인증 신청이 완료되었습니다. 검토까지 1-2일 소요됩니다" 토스트
   * 상태 → 인증 대기로 변경
* 인증 반려 상태: [재신청하기] 버튼으로 재제출 가능
⚠️ 에러 처리:
* 파일 크기 초과: "파일 크기는 10MB 이하만 가능합니다" 토스트
* 지원하지 않는 파일 형식: "JPG, PNG, PDF 파일만 업로드 가능합니다" 토스트
* 업로드 실패: "파일 업로드에 실패했습니다" 토스트
________________


P2-9. 학원 프로필 조회
🎯 페이지 목표: 지원 전 학원의 상세 정보를 확인
📐 레이아웃:
* 상단 헤더: 뒤로가기, 학원명
* 학원 기본 정보:
   * 대표 이미지 (있는 경우)
   * 학원명, 인증 배지
   * 평균 평점 및 리뷰 수
   * 주소, 연락처
   * [위치 보기] 버튼
* 학원 소개 섹션
* 등록된 공고 목록 (해당 학원의 다른 공고)
* 리뷰 섹션:
   * 평점 분포 그래프
   * 리뷰 목록 (최신순, 3개 미리보기)
   * [전체 리뷰 보기] 버튼
🎨 디자인 상세:
* 평점: 큰 숫자 + 별 아이콘
* 리뷰 카드: 강사명, 평점, 작성일, 내용 요약
🔄 인터랙션:
* [위치 보기]: 지도 앱 연동 또는 확대 지도 모달
* 공고 카드 탭: 해당 공고 상세로 이동
* [전체 리뷰 보기]: P2-10 리뷰 목록 페이지로 이동
________________


P2-10. 학원 리뷰 목록
🎯 페이지 목표: 학원에 대한 모든 리뷰 조회
📐 레이아웃:
* 상단 헤더: 뒤로가기, "학원 리뷰"
* 평점 요약 (평균 평점, 총 리뷰 수)
* 정렬 옵션: 최신순, 평점 높은순, 평점 낮은순
* 리뷰 카드 목록:
   * 작성자명 (익명)
   * 평점 (별)
   * 작성일
   * 리뷰 내용
🔄 인터랙션:
* 정렬 변경: 목록 재정렬
* 무한 스크롤로 추가 리뷰 로드
🔍 빈 상태:
* "아직 작성된 리뷰가 없습니다"
________________


P2-11. 리뷰 작성
🎯 페이지 목표: 채용 확정된 학원에 대한 리뷰 작성
📐 레이아웃:
* 상단 헤더: 뒤로가기, "리뷰 작성", [완료] 버튼
* 학원 정보 요약 (학원명, 근무 기간)
* 평점 선택 (별 1-5개)
* 리뷰 내용 입력 (텍스트 영역, 최소 10자, 최대 500자)
* 안내 문구: "솔직하고 객관적인 리뷰를 부탁드립니다"
🎨 디자인 상세:
* 별 아이콘: 탭하여 평점 선택, 선택된 별은 Soft Coral
* 글자 수 카운터 표시
🔄 인터랙션:
* 별 탭: 해당 평점 선택
* [완료]: 평점 선택 + 최소 글자 수 충족 시 활성화
   * 완료 시 "리뷰가 등록되었습니다" 토스트
   * 마이페이지로 이동
* 뒤로가기 (작성 중): "작성 중인 리뷰가 있습니다" 확인 모달
⚠️ 에러 처리:
* 최소 글자 수 미달: "최소 10자 이상 입력해주세요" 에러 메시지
________________


P2-12. 내가 작성한 리뷰
🎯 페이지 목표: 내가 작성한 리뷰 목록 조회 및 관리
📐 레이아웃:

   * 평점
   * 리뷰 내용
   * 작성일
   * [수정] / [삭제] 버튼
🔄 인터랙션:
* [수정]: 리뷰 수정 페이지로 이동 (P2-11과 동일한 형태)
* [삭제]: "정말 삭제하시겠습니까?" 확인 모달
   * 삭제 시 "리뷰가 삭제되었습니다" 토스트
🔍 빈 상태:

P2-13. 알림 페이지
🎯 페이지 목표: 지원 결과, 인증 결과 등 모든 알림 확인
📐 레이아웃:
* 상단 헤더: 뒤로가기, "알림", [모두 읽음] 버튼
* 알림 카드 목록:
   * 알림 아이콘 (유형별)
   * 제목
   * 내용 요약
   * 시간
   * 읽음/안읽음 표시 (배경색 차이)
🎨 디자인 상세:
* 안읽은 알림: 연한 Serene Blue 배경
* 읽은 알림: 흰색 배경
* 알림 유형별 아이콘 색상:
   * 지원 관련: Serene Blue

   * 인증: Info
🔄 인터랙션:
* 알림 카드 탭: 관련 페이지로 이동 + 읽음 처리
* [모두 읽음]: 모든 알림 읽음 처리
* 좌측 스와이프: 삭제
🔍 빈 상태:
* "새로운 알림이 없습니다"
________________

* 토글 변경 시: 변경 사항 표시

   * 비밀번호 재입력 필요

P2-16. 고객센터
🎯 페이지 목표: 자주 묻는 질문 및 문의하기

P3. 학원 회원
P3-1. 구인 공고 등록
🎯 페이지 목표: 필수 정보를 누락 없이 정확하게 공고 작성
📐 레이아웃:
* 상단 헤더: 뒤로가기, "구인 공고 등록", [저장] 버튼
* 입력 필드:
   * 공고 제목* (예: "주말 발레 강사 모집")
   * 근무 지역* (시/구 선택)
   * 무용 장르* (다중 선택 칩)
   * 담당 수업* (예: 초등반, 성인취미반)
   * 근무 요일 및 시간*
      * 요일 선택 (다중)
      * 시간대 입력 (시작-종료)
   * 급여 정보*
      * 유형 선택 (시급/월급)
      * 금액 입력
   * 우대 사항 (선택)
   * 상세 설명 (텍스트 영역)
* 하단: [등록하기] 버튼
🎨 디자인 상세:
* 필수 항목 옆 빨간 별표(*) 표시
* 장르 선택: 칩 형태 다중 선택
* 요일 선택: 버튼 그룹 (다중 선택 가능)
* 필수 항목 미입력 시 해당 필드 Error 색상 강조
🔄 인터랙션:
* [등록하기]: 필수 항목 모두 입력 시 활성화
   * 등록 시 "관리자 검토 후 공고가 게시됩니다" 토스트
   * 공고 관리 페이지로 이동
* 뒤로가기 (작성 중): "작성 중인 공고가 있습니다" 확인 모달
* [저장]: 임시저장 기능 (작성 중 상태로 저장)
⚠️ 에러 처리:
* 필수 항목 누락: "필수 항목을 모두 입력해주세요" 토스트
* 급여 0원 입력: "올바른 급여를 입력해주세요" 에러 메시지
________________


P3-2. 공고 관리
🎯 페이지 목표: 등록한 공고 목록 조회 및 관리
📐 레이아웃:
* 상단 헤더: 로고, 새 공고 등록(+), 알림
* 상태별 탭: 전체, 게시중, 검토 대기, 마감
* 공고 카드 목록:
   * 공고 제목
   * 장르, 급여
   * 지원자 수 (배지)
   * 등록일
   * 상태 배지
   * [수정] / [마감] / [삭제] 버튼
* 하단 네비게이션 바
🎨 디자인 상세:
* 상태 배지:
   * 게시중: Success
   * 검토 대기: Warning
   * 마감: Disabled
* 지원자 수 배지: Soft Coral (0명이면 회색)
🔄 인터랙션:
* 카드 탭: 공고 상세 조회 (P3-4)
* [수정]: 공고 수정 페이지 (P3-1과 동일한 형태)
* [마감]: "공고를 마감하시겠습니까?" 확인 모달
   * 마감 시 상태 → 마감, 새 지원 불가
* [삭제]: "삭제된 공고는 복구할 수 없습니다" 확인 모달
* (+) 버튼: P3-1 공고 등록으로 이동
🔍 빈 상태:
* "등록된 공고가 없습니다" + [첫 공고 등록하기] 버튼
________________

P3-4. 지원자 관리
🎯 페이지 목표: 공고별 지원자를 효율적으로 검토 및 관리
📐 레이아웃:
* 상단 헤더: 로고, 알림
* 공고 선택 드롭다운 (게시중인 공고 목록)
* 상태별 탭: 전체, 새 지원, 검토중, 채용 확정, 불합격
* 지원자 카드 목록:
   * 프로필 사진
   * 이름, 인증 배지
   * 전문 분야 (장르)
   * 경력 요약
   * 지원일
   * 상태 배지
* 하단 네비게이션 바
🎨 디자인 상세:
* 새 지원: 카드 좌측에 빨간 점 표시
* 인증 배지: Success 색상으로 신뢰도 강조
🔄 인터랙션:
* 공고 드롭다운 변경: 해당 공고의 지원자만 필터링
* 탭 전환: 상태별 필터링
* 카드 탭: P3-6 지원자 상세 프로필로 이동
🔍 빈 상태:
* "아직 지원자가 없습니다"
________________


P3-5. 지원자 상세 프로필 조회
🎯 페이지 목표: 지원자의 상세 정보를 확인하여 채용 결정
📐 레이아웃:
* 상단 헤더: 뒤로가기, 지원자 이름
* 기본 정보:
   * 프로필 사진
   * 이름, 인증 배지
   * 연락처 (채용 확정 전에는 비공개/일부 가림 처리 가능)
   * 전문 분야
* 자기소개 섹션
* 경력 섹션 (카드 형태)
* 학력 섹션 (카드 형태)
* 평균 평점 및 리뷰 (있는 경우)
* 하단 고정 액션 버튼:
   * [불합격 처리] (Secondary, Error 색상)
   * [채용 확정] (Primary, Success 색상)
🎨 디자인 상세:
* 섹션별 명확한 구분
* 인증 완료 시 각 항목에 체크 아이콘 표시
🔄 인터랙션:
* [채용 확정]: "해당 강사를 최종 채용하시겠습니까?" 확인 모달
   * 확정 시 "채용이 확정되었습니다" 토스트
   * 지원자 상태 → 채용 확정
   * 강사에게 알림 발송
   * 리뷰 작성 가능 상태로 변경
   * 연락처 전체 공개
* [불합격 처리]: "불합격 처리하시겠습니까?" 확인 모달
   * 처리 시 상태 → 불합격
   * 강사에게 알림 발송
________________

P4. 관리자 (웹 기반)
P4-1. 관리자 대시보드
🎯 페이지 목표: 서비스 전반의 현황을 한눈에 파악
📐 레이아웃 (2단 구조):
* 좌측: 측면 네비게이션
* 우측 메인 콘텐츠:
   * 통계 카드 (4개):
      * 총 회원 수 (강사/학원)
      * 인증 대기 건수
      * 게시중 공고 수
      * 신규 가입 (금주)
   * 최근 활동 타임라인
   * 빠른 액션 버튼:
      * 인증 검토하기
      * 회원 관리
      * 공고 관리
🎨 디자인 상세:
* 통계 카드: 숫자 크게, 전주 대비 증감 표시
* 타임라인: 시간순 정렬
🔄 인터랙션:
* 통계 카드 클릭: 해당 관리 페이지로 이동
* 인증 대기 건수 배지: 건수 있으면 Soft Coral로 강조
________________


P4-2. 회원 통합 관리
🎯 페이지 목표: 모든 회원을 효율적으로 관리
📐 레이아웃:
* 상단: 검색 및 필터 영역
   * 검색창 (이름, 이메일)
   * 필터: 회원 유형 (전체/강사/학원), 인증 상태, 계정 상태
* 회원 목록 테이블:
   * 컬럼: 이름, 유형, 이메일, 가입일, 인증 상태, 계정 상태, 액션
* 페이지네이션
🎨 디자인 상세:
* 테이블 헤더: 고정 (스크롤 시)
* 인증 상태: 배지로 시각화
* 계정 상태: 활성(Success) / 정지(Error)
🔄 인터랙션:
* 검색: 실시간 검색 (debounce)
* 필터 변경: 테이블 즉시 업데이트
* 이름 클릭: 회원 상세 정보 모달
* [계정 정지]: "정말 정지하시겠습니까?" 확인 모달
   * 정지 사유 입력 필요
* [계정 복구]: 즉시 활성 상태로 변경
________________


P4-3. 인증 서류 검토
🎯 페이지 목표: 제출된 인증 서류를 신속하고 정확하게 검토
📐 레이아웃:
* 좌측 사이드바:
   * 대기중인 서류 목록
   * 각 항목: 이름, 유형(강사/학원), 제출일
   * 대기 건수 배지
* 우측 메인 영역:
   * 신청자 정보 (이름, 유형, 제출일시)
   * 제출된 서류 이미지/PDF 뷰어
   * 확대/축소 컨트롤
   * 하단 액션 버튼:
      * [반려] (Error 색상)
      * [승인] (Success 색상)
🎨 디자인 상세:
* 좌측 목록: 선택된 항목 Serene Blue 배경
* 서류 뷰어: 큰 화면으로 선명하게 확인
* 대기 건수: 측면 네비게이션에도 배지로 표시
🔄 인터랙션:
* 목록 항목 클릭: 해당 서류 로드
* [승인]: "승인하시겠습니까?" 확인 모달
   * 승인 시 신청자에게 알림 발송
   * 다음 대기 서류 자동 로드
* [반려]: 반려 사유 입력 모달 필수
   * 반려 시 신청자에게 사유와 함께 알림 발송
   * 재신청 가능 상태로 변경
* 키보드 단축키: 좌우 화살표로 이전/다음 서류 이동
⚠️ 에러 처리:
* 서류 로딩 실패: "서류를 불러올 수 없습니다" 에러 메시지
________________


P4-4. 공고 통합 관리

📐 레이아웃:
* 상단: 검색 및 필터 영역
   * 검색창 (공고 제목, 학원명)
   * 필터: 상태 (전체/게시중/마감/숨김), 등록일 범위
* 공고 목록 테이블:
   * 컬럼: 공고 제목, 학원명, 장르, 급여, 등록일, 상태, 지원자 수, 액션
* 페이지네이션
🎨 디자인 상세:
* 상태별 배지 색상 구분
* 지원자 수: 숫자 배지로 표시
🔄 인터랙션:
* 공고 제목 클릭: 공고 상세 보기 모달
* [공고 숨기기]: "게시를 중단하시겠습니까?" 확인 모달
   * 숨김 처리 시 노출 중단, 학원에 알림
   * 사유 입력 필요
* [삭제]: "삭제된 공고는 복구할 수 없습니다. 정말 삭제하시겠습니까?" 강력한 확인 모달
   * 비밀번호 재입력 필요
   * 삭제 시 관련 지원 데이터 처리 방안 안내

🎯 인터랙션 패턴 가이드
1. 로딩 상태
* 풀스크린 로딩: 초기 페이지 진입 시
* 인라인 로딩: 데이터 갱신 시 (예: 필터 변경)
* 버튼 로딩: 폼 제출 시 버튼에 스피너 표시
* 스켈레톤: 카드 목록 초기 로딩 시
2. 확인 모달 우선순위
* 일반 확인: 취소 가능한 액션 (예: 지원하기)
* 주의 확인: 되돌리기 어려운 액션 (예: 공고 마감)
* 강력한 확인: 복구 불가능한 액션 (예: 삭제, 탈퇴)
   * 비밀번호 재입력 또는 확인 문구 입력 필요
3. 폼 유효성 검증
* 실시간 검증: 이메일 형식, 중복 확인
* 제출 시 검증: 모든 필수 항목 확인
* 에러 표시: 해당 필드 하단에 에러 메시지
* 성공 표시: Success 색상 테두리 또는 체크 아이콘
4. 리스트 인터랙션
* 탭: 상세 페이지로 이동
* 스와이프 (모바일): 좌측 스와이프로 삭제/관리 액션 노출
* 길게 누르기: 추가 옵션 메뉴 (신고, 공유 등)
* 당겨서 새로고침: 목록 최신화
5. 검색 및 필터
* 검색: Debounce 500ms 적용, 실시간 결과 표시
* 필터: 선택 즉시 결과 반영
* 초기화: [필터 초기화] 버튼으로 한 번에 초기화
